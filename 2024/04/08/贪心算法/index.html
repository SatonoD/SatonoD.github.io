贪心算法

SatonoD Lv2
  2024-04-08 16:04:43 2024-04-08 16:04:43 Created   2024-04-15 09:10:39 2024-04-15 09:10:39 Updated  

贪心算法一般按如下步骤进行：
① 建立数学模型来描述问题
② 把求解的问题分成若干个子问题
③ 对每个子问题求解，得到子问题的局部最优解
④ 把子问题的解局部最优解合成原来解问题的一个解

贪心算法 + 单调栈问题

LC 402. 移掉 K 位数字

给你一个以字符串表示的非负整数 num 和一个整数 k ，移除这个数中的 k 位数字，使得剩下的数字最小。请你以字符串形式返回这个最小的数字。

本题贪心的策略很简单，很好想。根据数字的特性，一定是优先删除从左到右第一个开始递减的数字。即从第一个局部最大值开始删除。

官方解答给出了很好的表达

给定一个长度为 n 的数字序列 $$[ D_0, D_1, D_2, D_3, …,D_{n−1}]$$
，从左往右找到第一个位置 i（i>0） 使得 $$D_i<D_{i−1} ，并删去D_{i-1}$$
如果不存在，说明整个数字序列单调不降，删去最后一个数字即可。

直接暴力计算的方法很好想，但是，如果数列为单调不降的，时间复杂度将来到O（nk）！
那么我们也可以考虑每次要删除数字的时候，从上一个数字继续检查单调性就可以了。那么栈这一数据结构就非常适合我们的问题了！每到需要删除的时候，我们直接将其从栈中弹出，然后k值减一就可以实现了。
剩下的，就是注意删去前缀0的问题了

贪心算法 + 序列

LC 334. 递增的三元子序列

给你一个整数数组 nums ，判断这个数组中是否存在长度为 3 的递增子序列。
如果存在这样的三元组下标 (i, j, k) 且满足 i < j < k ，使得 nums[i] < nums[j] < nums[k] ，返回 true ；否则，返回 false 。

这个问题的话，我们可以从头遍历到尾。想象有三个指针first、second和third。

初始时，first = nums[0]，second = +∞。对于 1 ≤ i <n1，当遍历到下标 i 时，令 num = nums[i]，进行如下操作：
if num > second，则找到了一个递增的三元子序列，返回 true ；
elif num > first，则将 second 的值更新为 num；
else first 的值更新为 num。

如果遍历结束时没有找到递增的三元子序列，返回 false。

LC 55. 跳跃游戏

给你一个非负整数数组 nums ，你最初位于数组的 第一个下标 。数组中的每个元素代表你在该位置可以跳跃的最大长度。
判断你是否能够到达最后一个下标，如果可以，返回 true ；否则，返回 false 。

我们直接使用贪心算法，从左到右开始遍历数组，每当能更新到最远的位置，就把它记录下来，直到能够更新到最后一个数，或者遍历到的数已经超过了当前能更新的最远的位置！！！！
